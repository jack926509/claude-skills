# 📝 投資報告撰寫 Skill v2.0

## 🎯 Skill 目的
將市場分析、個股研究、技術面資料整合為專業、易讀、可對外分享的投資報告，適合個人筆記、社群分享或正式報告用途。

---

## 📋 觸發條件

```
「幫我寫一份 [主題] 的報告」
「整理成正式格式」
「生成每日投資摘要」
「把分析結果做成報告」
```

---

## 📄 報告類型與對應格式

| 類型 | 使用時機 | 篇幅 | 格式特色 |
|------|---------|------|---------|
| 每日市場快報 | 每日例行 | 600~1000字 | 數據導向、表格豐富 |
| 個股深度報告 | 重要持股 | 1500~3000字 | 分析深入、含評分 |
| 週度策略報告 | 每週 | 1000~2000字 | 回顧+展望+操作建議 |
| 主題研究報告 | 產業/趨勢 | 2000字以上 | 宏觀視角、長線布局 |

## 🎨 寫作風格規範

### 段落原則
- 每段 **不超過 5 行**，避免大段落
- 重點數據用 **粗體** 標示
- 核心結論用引用框 `>` 突顯
- 列點最多 3 層，每點 1-2 句

### 標題層級
```
# 一級（報告標題）
## 二級（主要章節）
### 三級（子分析項目）
#### 四級（細項，謹慎使用）
```

### 數字格式統一
```
股價：$XXX.XX（千位逗號）
百分比：+X.XX% / -X.XX%（正負號不省略）
成交量：X.XX 億股
市值：$X.XXX 兆 / X,XXX 億
```

### 語氣要求
- 專業客觀，避免主觀情緒
- 用「可能」「傾向」「預期」取代絕對化語言
- 觀點要有數據或邏輯支撐
- 避免重複使用同一詞彙（如「大幅」「顯著」）

## 🔄 報告生成流程

```
Step 1：確認報告類型 → 選擇對應格式
Step 2：蒐集所需數據（先執行 stock-analysis 或 technical-analysis skill）
Step 3：撰寫執行摘要（最重要，先寫）
Step 4：展開各章節內容
Step 5：檢查品質清單
Step 6：加入免責聲明
```

---

## ✅ 品質檢查清單

- [ ] 執行摘要包含 3-5 個重點
- [ ] 所有數字來源明確
- [ ] 標題層級不超過 3 層
- [ ] 核心結論使用引用框突顯
- [ ] 語氣客觀，無絕對化語言
- [ ] 段落清晰，每段不超過 5 行
- [ ] 免責聲明已附上
- [ ] 報告總字數符合類型要求
